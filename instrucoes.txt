INSTRUCOES DO PROJETO

1. GITIGNORE
Antes de subir o projeto corretamente no GitHub, gere um .gitignore.
Entre nesse site: 
https://www.toptal.com/developers/gitignore
Digite django e copie o texto todo que mostrar e cole no arquivo .gitignore
Esse arquivo deve esta na pasta raiz do projeto(acad_time), na mesma pasta que esta as pastas backend, 
frontend e etc

2. ESTRUTURA DO PROJETO
A pasta backend contém a API em Django.
A pasta frontend contém o front separado com HTML, CSS e JavaScript.

3. VENV
Cada integrante precisa criar ou usar a própria venv localmente dentro da pasta backend.
No terminal entre na pasta do backend:
    cd backend
Crie a venv local com esse comando
    python -m venv venv

4. ATIVAR A VENV
No PowerShell:
cd backend (precisa está na pasta backend)
    .\venv\Scripts\Activate.ps1

5. INSTALAR AS DEPENDENCIAS
Ainda dentro da pasta backend:
no terminal:
    pip install -r requirements.txt

6. ARQUIVO .ENV
Dentro da pasta backend precisa criar um arquivo .env
Esse arquivo precisa ser ajustado antes de rodar o projeto.
vc primeiro precisa entrar no terminal, dentro da pasta backend rodar o comando:
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

ele vai gerar a chave secreta que vai ser usada dentro do arquivo .env
Arquivo .env

SECRET_KEY= (chave secreta aqui)
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost

DATABASE_URL= 

Em database_url vcs precisam me perguntar a url do banco


7. RODAR O BACKEND
Depois de configurar o .env:
(primeira vez, depois apenas se modificar o banco)
Terminal dentro de backend:
    python manage.py migrate

8. Sempre que quiser a API funcionando
Terminal dentro de backend:
    python manage.py runserver 7000

19. RODAR O FRONTEND
A pasta frontend pode ser aberta com Live Server no VS Code, ou outra forma de servidor local.
